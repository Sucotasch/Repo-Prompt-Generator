## 🎯 О проекте

**Repo-Prompt-Generator** — это интеллектуальный инструмент для разработчиков, который использует искусственный интеллект для анализа кодовых баз и генерации структурированных промптов, технической документации и аудитов безопасности.

Приложение поддерживает работу как с **GitHub репозиториями**, так и с **локальными файлами**, используя передовые технологии RAG (Retrieval-Augmented Generation) и гибридного поиска для точного анализа кода.

---

## ⚡ Возможности

1. **Генерация промптов для AI-ассистентов** (Gemini CLI, Cursor, Claude, etc.)
2. **Аудит кодовой базы** (архитектура, безопасность, производительность)
3. **Документирование** (генерация Markdown и диаграмм Mermaid)
4. **Интеграционный анализ** (сравнение репозиториев)
5. **Топологический RAG** (построение графа зависимостей кода для лучшего контекста)
6. **Мультипровайдер** (Gemini, Ollama, Qwen, OpenAI-совместимые API)

## 🏗️ Архитектура (Монорепозиторий)

Проект организован как монорепозиторий (npm workspaces), разделяющий общую логику и UI между веб-версией и десктопным приложением.

```text
repo-prompt-generator/
├── apps/
│   ├── web/          # Веб-версия (React + Vite + Express proxy)
│   └── desktop/      # Десктоп-версия (Tauri + Rust + React)
├── packages/
│   ├── core/         # Общая бизнес-логика, AI-сервисы, RAG, шаблоны
│   └── ui/           # Общие React-компоненты (Tailwind v4)
```

## 🖥️ Web vs Desktop (Tauri)

Приложение поставляется в двух вариантах, каждый из которых имеет свои особенности:

| Функция | Web-версия (`apps/web`) | Desktop-версия (`apps/desktop`) |
|---------|-------------------------|---------------------------------|
| **Среда выполнения** | Браузер + Node.js (Express) | Нативное приложение (Tauri/Rust) |
| **Доступ к локальным файлам** | Ограничен (только через drag & drop / выбор папки) | **Полный** (нативное чтение ФС через Rust) |
| **Обход CORS** | Через прокси-сервер (Express) | Напрямую из Rust-бэкенда (нет проблем с CORS) |
| **Управление Ollama** | Требует ручного запуска Ollama с флагом `OLLAMA_ORIGINS` | **Автоматическое** (запуск/остановка процесса из UI) |
| **Qwen Аутентификация** | Требует проксирования через бэкенд | Нативная обработка OAuth Device Flow |
| **Кэширование RAG** | IndexedDB / LocalStorage (лимиты браузера) | В перспективе: SQLite / Нативная ФС |
| **Развертывание** | Cloud Run, Vercel, Docker | Установщики (.exe, .dmg, .deb) |

## 🛠️ Установка и запуск

### Требования

- **Node.js**: 20.x LTS или новее
- **npm**: 10.x или новее
- **Rust** (только для Desktop-версии): `rustc` и `cargo` (установка через [rustup](https://rustup.rs/))

### Шаг 1: Клонирование и установка

```bash
git clone https://github.com/Sucotasch/Repo-Prompt-Generator.git
cd Repo-Prompt-Generator
npm install
```

### Шаг 2: Настройка переменных окружения

Создайте файл `.env.local` в корне проекта (или скопируйте `.env.example`):

```env
# Gemini API (Опционально, если используете только Ollama)
GEMINI_API_KEY=your_gemini_api_key_here

# Ollama (Для полностью локальной работы)
OLLAMA_URL=http://localhost:11434
OLLAMA_MODEL=llama3.2
```

### Шаг 3: Запуск

**Запуск Web-версии (Браузер + Express):**
```bash
npm run dev
# Откроется на http://localhost:3000
```

**Запуск Desktop-версии (Tauri):**
```bash
npm run tauri dev -w apps/desktop
```

### Сборка для продакшена

```bash
# Сборка всех пакетов и веб-версии
npm run build

# Сборка исполняемого файла Tauri (Windows/macOS/Linux)
npm run tauri build -w apps/desktop
```

### Решение проблем (Desktop)

- **Отсутствуют иконки при сборке**: Убедитесь, что вы сгенерировали иконки с помощью команды `npx tauri icon -w apps/desktop` (иконки уже включены в репозиторий, но если вы их изменили, потребуется перегенерация).
- **Ошибки CORS в Desktop-версии**: В актуальной версии Tauri-приложения все запросы к Ollama и Qwen выполняются через нативный Rust-бэкенд, поэтому проблемы с CORS исключены. Если вы видите предупреждение о CORS, убедитесь, что вы запустили именно Desktop-версию, а не Web-версию в браузере.

## 📖 Использование

1. **Выберите источник кода**: Вставьте ссылку на GitHub репозиторий или выберите локальную папку.
2. **Выберите AI-провайдера**: Gemini (требует API ключ), Ollama (локально, бесплатно), Qwen или кастомный OpenAI-совместимый эндпоинт.
3. **Выберите шаблон**: Architecture, Security, Audit, Docs и т.д.
4. **Настройте RAG (опционально)**: В разделе "Advanced RAG Settings" можно настроить баланс гибридного поиска (BM25 vs Векторный) и размер чанков.
5. **Сгенерируйте результат**: Нажмите "Generate Prompt". Результат можно скопировать или скачать в виде Markdown-файла.

## 📐 Шаблоны анализа

- **Default**: Универсальный системный промпт для анализа кодовой базы.
- **Architecture**: Глубокий анализ архитектуры с генерацией Mermaid-диаграмм.
- **Audit**: Поиск дефектов, антипаттернов и узких мест в производительности.
- **Security**: Аудит уязвимостей (OWASP) и угроз безопасности.
- **Docs**: Автоматическая генерация технической документации по API и модулям.
- **Integration**: Анализ совместимости двух репозиториев (требует указания Reference URL).
- **ELI5**: Простое объяснение сложных концепций кода для новичков.

## 🧰 Технологический стек

- **Монорепозиторий**: npm workspaces
- **Frontend**: React 19, Vite 6, Tailwind CSS v4, Framer Motion, Lucide React
- **Backend (Web)**: Express, Node.js
- **Backend (Desktop)**: Rust, Tauri 2.0
- **AI Интеграции**: `@google/genai`, Ollama API, Qwen API
- **RAG Engine**: Кастомная реализация гибридного поиска (BM25 + Cosine Similarity + Reciprocal Rank Fusion)
