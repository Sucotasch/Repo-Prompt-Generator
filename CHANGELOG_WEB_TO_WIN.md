# Журнал изменений (Web -> Windows)

Этот документ предназначен для отслеживания изменений в веб-версии проекта, которые необходимо перенести в локальную Windows-версию (Tauri).

## [2026-03-17] - Отделение веб-версии от Tauri
### Изменено
- Удалены директории `src-tauri` и `tauri_branch_temp`, чтобы они не попадали в репозиторий GitHub.
- Из `package.json` удалены все зависимости `@tauri-apps/*`.
- В файлах `src/utils/fileSystem.ts` и `src/utils/tauriFetch.ts` удалены динамические импорты плагинов Tauri. Теперь они используют стандартные веб-API (`fetch` и `Blob`/`URL.createObjectURL`).

*Примечание для локальной Windows-версии:* При переносе будущих изменений из этого журнала, игнорируйте изменения, связанные с удалением Tauri-специфичного кода, так как в локальной версии он должен остаться.

## [2026-03-17] - Оптимизация RAG-запросов и кэширование
### Изменено
- В `src/App.tsx` добавлена логика пропуска LLM-экспансии (Query Expansion) для стандартных шаблонов, если поле RAG-запроса пустое. Теперь сразу используется `defaultSearchQuery` шаблона.
- Добавлено кэширование в `localStorage` для сгенерированных RAG-запросов кастомных шаблонов, чтобы избежать повторных обращений к LLM.
- В `src/App.tsx` (функция `getFallbackQuery`) добавлено включение `additionalContext` (Task Instructions & External Context) и имен прикрепленных файлов (`attachedDocs`) в базовый RAG-запрос. Если эти поля заполнены, пропуск LLM-экспансии отменяется, и запрос специализируется под конкретную задачу пользователя.

## [2026-03-20] - Глобальное обновление RAG: Гибридный поиск и Кэширование эмбеддингов
### Архитектурные изменения и инструкции для миграции
- **Кэширование эмбеддингов (`src/services/embeddingCacheService.ts`)**: Реализовано локальное сохранение векторов файлов. Это критически снижает затраты на API и ускоряет повторные запросы.
  * *💡 Заметка для Tauri:* В веб-версии кэш опирается на браузерные квоты (IndexedDB/localStorage). При переносе в Windows-версию **настоятельно рекомендуется** переписать слой хранения на нативную файловую систему Tauri (`@tauri-apps/plugin-fs`) или локальный SQLite. Десктопные репозитории могут быть огромными, и WebView быстро упрется в лимиты памяти браузера.
- **Гибридный поиск (`src/services/ragService.ts`)**: Поиск переписан с чисто векторного на гибридный (Векторы + BM25 + Reciprocal Rank Fusion). Теперь система ищет как по смыслу, так и по точному совпадению терминов.
  * *💡 Заметка для Tauri:* Вся тяжелая математика (TF-IDF для BM25, косинусное сходство) сейчас крутится в JS-потоке. Для первой итерации миграции это можно перенести "как есть". Однако архитектурно это идеальный кандидат для выноса в Rust-бэкенд Tauri (через `invoke`), что даст колоссальный прирост производительности при поиске по локальным проектам на 10 000+ файлов.
- **UI расширенных настроек (Advanced RAG Settings)**: В интерфейс добавлены контролы для `topK` (количество чанков), `searchStrategy` (баланс Вектор/BM25) и кнопка ручной очистки кэша.
  * *💡 Заметка для Tauri:* Это чистый React-код, переносится 1:1 без каких-либо адаптаций.

## [2026-03-21] - Изоляция и умная очистка кэша эмбеддингов
### Добавлено / Изменено
- В `src/services/embeddingCacheService.ts` добавлена функция `pruneUnusedCaches` с 5-минутным "льготным периодом" (grace period). Она автоматически удаляет кэш векторов для репозиториев, которые не используются в данный момент, защищая от раздувания памяти браузера.
- В `src/services/ragService.ts` функции `getEmbedding` и `performRAG` теперь принимают `repoUrl` для строгой изоляции кэша по пространствам имен (namespaces) каждого репозитория.
- В `src/App.tsx` добавлен `useEffect`, который отслеживает изменение `url` и `referenceUrl` и триггерит хирургическую очистку неиспользуемых кэшей.

## [2026-03-23] - Внедрение графа зависимостей (Topology-Aware RAG)
### Добавлено / Изменено
- Создан новый файл `src/utils/codeGraph.ts` с утилитой `buildCodeDependencyGraph`. Она парсит внутренние импорты (ES6/CommonJS) и строит текстовый граф связей в формате Graphviz DOT (с жестким лимитом в 3000 символов для экономии контекста).
- В `src/services/geminiService.ts` (функция `buildPromptText`) добавлена инъекция сгенерированного DOT-графа в финальный системный промпт. Теперь LLM видит не только дерево файлов, но и их логические связи (архитектурную топологию) как для целевого, так и для референсного репозиториев.

## [2026-03-25] - Подготовка к Монорепозиторию и интеграция системных фич Windows
### Архитектурные планы и инструкции для миграции
- **Стратегия Монорепозитория (Single Source of Truth)**: Принято решение не поддерживать две раздельные кодовые базы. Текущая (самая свежая) веб-версия (`src`) станет ядром (`packages/core` и `packages/ui`). Это гарантирует, что все новые фичи (настройки RAG, новые модели, шаблоны) будут автоматически доступны в обеих версиях.
- **Паттерн Адаптер (Dependency Injection)**: Для изоляции платформозависимого кода (Node.js/Express для веба vs Rust/Tauri для десктопа) будет использоваться внедрение зависимостей. Пакет `core` будет описывать интерфейсы (например, `IPlatform`), а точки входа `apps/web` и `apps/desktop` будут передавать конкретные реализации.
- **Системные фичи из ветки `windows-tauri-final`**: При создании `apps/desktop` мы **полностью перенесем** Rust-код (`src-tauri/src/lib.rs` и `main.rs`) из ветки `windows-tauri-final`. Нас интересуют следующие критически важные реализации:
  * *Управление Ollama*: Запуск (`start_ollama`) и остановка (`stop_ollama`) процессов без мелькающих окон консоли (`CREATE_NO_WINDOW`).
  * *Graceful Shutdown*: Автоматическое завершение процессов `ollama.exe` при закрытии окна Tauri (перехват `tauri::RunEvent::Exit` и проверка `we_started_ollama`), чтобы не оставлять "зомби-процессы" в памяти.
  * *Поддержка Proxy*: Изолированные HTTP-клиенты на базе `isahc` с поддержкой проксирования через переменные окружения `HTTP_PROXY`/`HTTPS_PROXY`.
  * *Безопасные вызовы Gemini*: Выполнение запросов к Gemini API напрямую из Rust (`call_gemini_secure`), минуя браузерные ограничения CORS.
- **UI-компоненты**: Поля для ввода Proxy и кнопки управления локальной Ollama будут перенесены в общий UI (`packages/ui`) и будут активироваться/деактивироваться в зависимости от текущего адаптера платформы.
